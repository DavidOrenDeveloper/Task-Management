# הגדרת תזכורות בענן (חינמי לגמרי, בלי כרטיס אשראי)

זה מדריך חד-פעמי. אחרי שמסיימים אותו, תזכורות יתריעו בטלפון **גם כשהאפליקציה סגורה
לגמרי** — בדיוק כמו תזכורות רגילות של הטלפון.

איך זה עובד בקצרה: הנתונים של התזכורות (לא שום דבר אחר באפליקציה) נשמרים גם
ב-Firestore (מסד נתונים חינמי של Google), ו-GitHub Actions (שירות חינמי של
GitHub) בודק כל 10 דקות אם הגיע זמן של תזכורת כלשהי, ואם כן — שולח התראה אמיתית
לטלפון דרך שירות ה-Push המובנה של הדפדפן. זה בדיוק אותו מנגנון שאפליקציות
אמיתיות (כולל Samsung Reminders) עובדות איתו.

**חשוב:** בלי ההגדרה הזו, האפליקציה ממשיכה לעבוד בדיוק כמו שהיא עובדת היום —
תזכורות פשוט יתריעו רק כשהאפליקציה פתוחה/ברקע. אין חובה לבצע את המדריך הזה.

---

## איך מעלים את הקבצים ל-GitHub (כולל התיקייה היחידה שחייבת להיות בנתיב מסוים)

כל הקבצים בזיפ הזה, **חוץ מקובץ אחד**, נמצאים ישירות ברמה העליונה — פשוט גוררים
את כולם ביחד לריפו כמו תמיד.

הקובץ החריג הוא `send-reminders.yml`, והוא **חייב** לשבת בנתיב
`.github/workflows/send-reminders.yml` — זו דרישה טכנית קשיחה של GitHub עצמו
(ככה הוא מזהה שיש לו "פעולה" להריץ), אי אפשר לעקוף אותה. אבל אין צורך "להעלות
תיקייה" בשביל זה — הכי קל דרך האתר של GitHub:

1. בריפו ← **Add file → Create new file**.
2. בתיבת שם הקובץ, מקלידים בדיוק: `.github/workflows/send-reminders.yml`
   (עם הלוכסנים) — GitHub יוצר את שתי התיקיות לבד ברגע שרואה `/` בשם.
3. פותחים את הקובץ `send-reminders.yml` שבזיפ בעורך טקסט, מעתיקים את כל
   התוכן, ומדביקים בעורך של GitHub.
4. **Commit new file**.

זהו — זה הקובץ היחיד שדורש טיפול מיוחד. כל שאר הקבצים (כולל `send-reminders.js`)
נמצאים ברמה הראשית של הזיפ ועולים כרגיל.

## שלב 1 — יצירת פרויקט Firebase (חינמי, Google בלבד, בלי כרטיס אשראי)

1. כנסו ל-[console.firebase.google.com](https://console.firebase.google.com) עם חשבון Google.
2. "Add project" / "הוספת פרויקט" → תנו שם (למשל `electrician-site-app`) → אפשר
   לכבות את Google Analytics (לא נחוץ כאן) → "Create project".
3. בתפריט הצד: **Build → Authentication → Get started → Sign-in method** →
   בחרו **Anonymous** → Enable → Save.
   (זה מה שנותן לאפליקציה "כניסה שקטה" בלי שום מסך התחברות שאתם רואים.)
4. בתפריט הצד: **Build → Firestore Database → Create database** → בחרו מיקום
   קרוב (למשל `eur3 (europe-west)`) → Start in **production mode** → Enable.
5. אחרי שנוצר: לשונית **Rules** → מחקו את התוכן שיש שם → הדביקו את התוכן המלא
   של הקובץ `firestore.rules` שצורף כאן → **Publish**.

## שלב 2 — הוספת "אפליקציית Web" וקבלת הקונפיגורציה

1. בדף הראשי של הפרויקט (⚙️ ליד "Project Overview") → **Project settings**.
2. גללו למטה ל"Your apps" → לחצו על סמל ה-`</>` (Web) → תנו שם → **Register app**
   (אין צורך ב-Firebase Hosting, אפשר לדלג).
3. תקבלו קטע קוד עם אובייקט בשם `firebaseConfig` שנראה כך:
   ```js
   const firebaseConfig = {
     apiKey: "AIza...",
     authDomain: "your-project.firebaseapp.com",
     projectId: "your-project",
     storageBucket: "your-project.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123456789:web:abcdef"
   };
   ```
4. פתחו את הקובץ `firebase-config.js` באפליקציה, והעתיקו את הערכים האלה
   לתוך `FIREBASE_CONFIG` (במקום `"YOUR_API_KEY"` וכו').

## שלב 3 — מפתחות VAPID (לזיהוי האפליקציה מול שירות ה-Push)

יצרתי לכם כבר זוג מפתחות תקין ומוכן לשימוש (נוצר עכשיו, לא נשלח לשום מקום):

```
VAPID_PUBLIC_KEY=BBXF7PvlWgU44COZ_or03Zo4smtU0ho8M1qIoQanQeZ0YOjKtXvW719cCTnoHdZSziNo8JwM5s06PC0s7NE_F5o
VAPID_PRIVATE_KEY=XD1DTb36yZOYnxeJcyTGl0OxffqErFOJawlhdzJiYVQ
```

- את ה-`VAPID_PUBLIC_KEY` יש להדביק בקובץ `firebase-config.js`.
- את שניהם (public + private) נדביק כ-GitHub Secrets בשלב 5.
- (אם תעדיפו זוג משלכם: אפשר ליצור חינמי וללא הרשמה באתר
  [vapidkeys.com](https://vapidkeys.com), הכל רץ בדפדפן בלבד.)

## שלב 4 — קובץ Service Account (כדי שה-GitHub Action יוכל לקרוא מ-Firestore)

1. Firebase console → ⚙️ **Project settings → Service accounts**.
2. **Generate new private key** → מאשרים → מוריד קובץ `.json` למחשב.
3. **חשוב:** לא מעלים את הקובץ הזה לשום מקום בריפו! הוא ישמש רק כ-Secret
   (השלב הבא). מי שיש לו את הקובץ הזה יכול לקרוא/לכתוב לפרויקט ה-Firebase שלכם.

## שלב 5 — הגדרת Secrets ב-GitHub

בריפו של האפליקציה: **Settings → Secrets and variables → Actions → New
repository secret**, ומוסיפים אחד-אחד (השם חייב להיות מדויק):

| שם ה-Secret               | ערך                                                        |
|----------------------------|-------------------------------------------------------------|
| `FIREBASE_SERVICE_ACCOUNT` | כל תוכן קובץ ה-JSON מ**שלב 4** — מדביקים כמות שהוא          |
| `SITE_ID`                  | אותו ערך בדיוק שיש ב-`SITE_ID` בתוך `firebase-config.js`     |
| `VAPID_PUBLIC_KEY`         | מ**שלב 3**                                                   |
| `VAPID_PRIVATE_KEY`        | מ**שלב 3**                                                   |
| `VAPID_SUBJECT`            | `mailto:` + כתובת האימייל שלכם (למשל `mailto:me@gmail.com`) |

## שלב 6 — הפיכת הריפו לציבורי (מומלץ)

GitHub Actions בריפו **פרטי** מוגבל ל-2000 דקות חינם בחודש — הרצה כל 10 דקות
עלולה לחרוג מזה. בריפו **ציבורי** Actions חינמי וללא הגבלה.

זה בטוח לחלוטין כאן כי שום סוד לא נמצא בקוד עצמו — `firebase-config.js` מכיל
רק ערכים ציבוריים (ככה Firebase Web תמיד עובד, ההגנה היא בחוקי ה-Firestore
Rules ולא בהסתרת הקובץ), וכל הסודות האמיתיים חיים אך ורק ב-GitHub Secrets.

**Settings → General → Danger Zone → Change visibility → Public.**

לא בנוח שהקוד יהיה ציבורי? אפשר להשאיר את הריפו פרטי ובמקום זה לערוך את
`.github/workflows/send-reminders.yml` ולשנות את השורה
`cron: "*/10 * * * *"` לתדירות נמוכה יותר, למשל כל 20 דקות: `"*/20 * * * *"`.

## שלב 7 — חיבור הטלפון

1. פתחו את האפליקציה המעודכנת בטלפון.
2. "עוד" ← **"התראות בטלפון"** ← אשרו.
3. מיד אחרי זה **"תזכורות בענן"** אמור לעבור אוטומטית ל-"✅ מחובר" (אם לא —
   לחצו עליו ידנית).
4. זה שולח ל-Firestore את פרטי המכשיר, כדי שה-GitHub Action ידע לאן לשלוח
   התראות.

## שלב 8 — בדיקה

1. צרו משימה עם תזכורת לעוד 2-3 דקות.
2. בגיטהאב: לשונית **Actions → Send due reminders → Run workflow** — כך
   אפשר להריץ ידנית בלי לחכות ל-cron, ולראות בלוג אם ההתראה נשלחה בהצלחה.
3. סגרו לגמרי את האפליקציה/דפדפן בטלפון, וודאו שההתראה בכל זאת מגיעה.

## דברים שכדאי לדעת

- **דיוק בזמן:** GitHub לא מתחייב לדיוק מוחלט בתזמון ה-cron — לפעמים יש איחור
  של כמה דקות בעומס. מספיק טוב לתזכורות אישיות, לא מדויק לשנייה.
- **iPhone:** תזכורות PWA דורשות iOS 16.4+ **וגם** שהאפליקציה תותקן דרך
  "שיתוף ← הוסף למסך הבית" (לא רק פתיחה בספארי).
- **בלי אינטרנט:** אם יוצרים תזכורת כשהטלפון במצב אופליין, הסנכרון לענן פשוט
  יתעכב עד לחיבור הבא — התזכורת המקומית (כשהאפליקציה פתוחה) עדיין תעבוד כרגיל.
- אפשר תמיד לדלג על כל המדריך הזה — האפליקציה עובדת מצוין גם בלעדיו, רק
  שתזכורות יתריעו אז רק כשהיא פתוחה/ברקע.
