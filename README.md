# ניהול אתר – קבלן חשמל

## מה חדש בגרסה 2.0.0

- **הערות מיקום** — בכל בניין/קומה אפשר לפתוח דף מיקום עם הערות, משימות והזמנות ששייכות אליו (לחיצה על שם הקומה, או "הערות ומשימות כלליות לבניין").
- **ניהול קטגוריות תוך כדי יצירת משימה/הזמנה** — אפשר להוסיף/להסיר סוגי משימות וקטגוריות הזמנה ישירות מתוך הטופס. במחיקת קטגוריה שיש בה פריטים, האפליקציה שואלת אם להעביר אותם ל"כללי" או למחוק גם אותם.
- **לחיצה ארוכה לסידור מחדש** — כמעט בכל רשימה (משימות, הזמנות, שאלות, בניינים, קומות, הערות) אפשר ללחוץ לחיצה ארוכה ולגרור לסדר מחדש.
- **חיפוש חכם** — כפתור החיפוש למעלה פותח חיפוש חי שמחפש בו-זמנית במשימות, הזמנות, שאלות, בניינים/קומות והערות, עם הדגשת התוצאות.
- **מיון** — כפתור "מיון" בכל רשימה מאפשר למיין לפי תאריך, א-ב, דחיפות, סטטוס או סדר ידני.
- **חלונות מותאמים** — כל האישורים/הודעות (כמו הוספת קומה) עברו מהתראות דפדפן פשוטות לחלונות מעוצבים בתוך האפליקציה.
- **תזכורות ("נודניקים")** — למשימה אפשר להגדיר תאריך/שעה + התראה חוזרת (כל 15 דק'/חצי שעה/שעה/יום). על כל משימה עם תזכורת מופיעים כפתורי "הושלם", "נודניק" (דחייה) ו"המתנה" (משהה את ההתראות בלי לסמן כהושלם ובלי למחוק). כרטיסים עם תזכורת שעברה מסומנים באדום, ותזכורות קרובות מסומנות בכתום, כולל פס התראה למעלה.
  - **חשוב לדעת:** האפליקציה רצה בדפדפן בלבד (בלי שרת), ולכן ההתראות פועלות כשהיא פתוחה/ברקע בדפדפן. זה לא זהה לתזכורות מובנות של הטלפון (כמו Samsung Reminders) שיכולות להתריע גם כשהאפליקציה סגורה לגמרי — זה מצריך שרת/שירות רקע של המערכת, מה שמעבר למה שאפשרי באפליקציית PWA טהורה בלי שרת. מומלץ לאשר התראות (בעמוד "עוד") ולא לסגור את האפליקציה לגמרי כשיש תזכורות פתוחות.

## רוצה שתזכורות יתריעו גם כשהאפליקציה סגורה לגמרי?

אפשר להוסיף חיבור חינמי לגמרי (Firebase + GitHub Actions, בלי כרטיס אשראי) שהופך את זה לאפשרי — ממש כמו תזכורת רגילה של הטלפון. זה לא חובה, האפליקציה עובדת מצוין גם בלעדיו. ההוראות המלאות, שלב-אחר-שלב, נמצאות בקובץ **`SETUP-CLOUD.md`** שצורף כאן.



אפליקציית PWA לניהול משימות, הזמנות, בניינים/קומות, שאלות והערות באתר בנייה.
כל הנתונים נשמרים **מקומית בטלפון** (אין שרת, אין צורך באינטרנט לשימוש שוטף).

## התקנה בטלפון (חד פעמי) — דרך GitHub Pages

1. היכנס ל-github.com וצור **Repository חדש** (ציבורי), למשל בשם `electrician-app`.
2. העלה את **כל הקבצים** (כולם באותה רמה, אין תיקיות) לתוך ה-repository.
   - בעמוד ה-repository לחץ "Add file" → "Upload files", גרור את **כל הקבצים ביחד** (index.html, style.css, app.js, storage.js, manifest.json, sw.js, האייקונים וה-README), ולחץ Commit.
   - חשוב: אל תעלה קובץ ZIP עצמו — קודם חלץ (extract) אותו במחשב, ואז גרור את הקבצים שבפנים.
3. בתוך ה-repository: **Settings** → בתפריט הצד **Pages**.
4. תחת "Build and deployment" → Source: בחר **Deploy from a branch**, Branch: **main**, Folder: **/(root)**. לחץ Save.
5. חכה כדקה-שתיים. בעמוד ה-Pages תופיע כתובת בסגנון:
   `https://USERNAME.github.io/electrician-app/`
6. פתח את הכתובת הזו **בטלפון**, בדפדפן (Chrome באנדרואיד / Safari באייפון).
7. **אנדרואיד (Chrome):** יופיע באנר "הוספה למסך הבית" — לחץ עליו. אם לא מופיע, לחץ על תפריט שלוש הנקודות ⋮ → "הוספה למסך הבית".
8. **iPhone (Safari):** לחץ על כפתור השיתוף (□↑) → "הוספה למסך הבית".
9. האפליקציה תופיע כאייקון רגיל במסך הבית, כולל פתיחה במסך מלא (בלי סרגלי דפדפן).

מרגע זה — האפליקציה עובדת **גם בלי אינטרנט** (חוץ מהבדיקה האוטומטית לעדכון גרסה, שרק בודקת ולא מפריעה אם אין רשת).

## איך מעדכנים את האפליקציה בעתיד

1. עדכן את הקבצים הרצויים (למשל דרך Claude, או ידנית ב-GitHub).
2. **חשוב:** בקובץ `sw.js`, שנה את השורה:
   ```js
   const CACHE_VERSION = "v1.0.0";
   ```
   לכל מספר גרסה חדש, לדוגמה `"v1.0.1"`. **בלי זה הטלפונים לא יזהו שיש עדכון.**
3. אפשר גם לעדכן את `APP_VERSION` בתחילת `js/app.js` (מוצג במסך "עוד").
4. העלה (Commit) את הקבצים המעודכנים ל-GitHub.
5. בפעם הבאה שתפתח את האפליקציה בטלפון, היא תבדוק אוטומטית ותציג באנר צהוב
   "🔄 גרסה חדשה זמינה" — לחיצה על "עדכן עכשיו" תרענן לגרסה החדשה.

## סנכרון בין כמה מכשירים (טלפון + מחשב וכו')

האפליקציה שומרת הכל מקומית בכל מכשיר בנפרד — **אין סנכרון אוטומטי בין מכשירים**.
כדי להעביר את הנתונים בין מכשירים, יש בתפריט **עוד** → **נתונים וגיבוי**:

- **ייצוא גיבוי (קובץ)** — מוריד קובץ JSON עם כל הנתונים. שלח אותו לעצמך (וואטסאפ/מייל) למכשיר השני.
- **ייבוא / שחזור גיבוי** — במכשיר השני, בחר את הקובץ שקיבלת. שים לב: הפעולה **מחליפה** את כל הנתונים הקיימים באותו מכשיר.

מומלץ להשתמש בטלפון כמכשיר הראשי (שם מוסיפים משימות בשטח), ולייצא מדי פעם גיבוי גם למחשב.

> **טיפ להמשך:** אם בעתיד תרצה סנכרון אוטומטי אמיתי בזמן אמת בין הטלפון למחשב (בלי ייצוא/ייבוא ידני), זה אפשרי בשלב הבא על ידי חיבור מסד נתונים חינמי כמו Firebase — זו תוספת שאפשר לבנות מאוחר יותר בלי לשנות את שאר האפליקציה.
